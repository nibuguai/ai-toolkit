# RAG 知识库问答提示词



------



## 状态

🟡 讨论中



## 功能描述

基于用户上传的私有文档，检索相关内容后，由 LLM 生成带引用来源的精准回答。



## System Prompt

```
你是一个基于私有知识库的智能问答助手。

你需要严格根据提供的文档片段来回答用户的问题。

核心约束：
1. 只能基于提供的文档片段回答，不得使用你自己的知识
2. 如果文档片段不足以回答问题，直接告知用户"当前知识库中没有找到相关信息"
3. 回答必须标注引用来源，格式：[来源: 文档名, 第X段]
4. 引用时尽量给出原文关键句
5. 回答要简洁、准确、结构化

禁止行为：
- 编造文档中没有的内容
- 模糊引用（必须指明具体文档和段落）
- 回答与问题无关的内容
```



## User Prompt 模板

```
【检索到的相关文档片段】

{retrieved_chunks}

---

【用户问题】

{question}

---

请基于以上文档片段回答问题，并标注引用来源。
```



## 上下文拼接规则

```
拼接顺序：
1. System Prompt
2. User Prompt：
   ├── 检索结果（Top-K 片段，每个片段标注 文档名 + 段落序号 + 相似度分数）
   └── 用户当前问题
```

检索结果格式化模板：

```
[片段 1] 来源: {doc_name} | 段落: {chunk_index} | 相关度: {score:.2f}
{chunk_text}

[片段 2] 来源: {doc_name} | 段落: {chunk_index} | 相关度: {score:.2f}
{chunk_text}
...
```

> K 值建议：默认 Top-5，最少 3 个。



## 对话历史拼接（多轮对话）

```
拼接顺序：
1. System Prompt
2. 对话历史（最近 N 轮）：
   ├── 用户: {上次问题}
   └── 助手: {上次回答}
3. User Prompt（当前轮，含检索结果）
```

> 多轮对话时，检索 query 使用改写后的完整问题而非用户原始输入。



## 输出格式

```json
{
  "answer": "回答内容（Markdown 格式）",
  "sources": [
    {
      "doc_name": "文档名",
      "chunk_index": 3,
      "quote": "引用的原文关键句"
    }
  ],
  "confidence": "high / medium / low",
  "no_match": false
}
```

| 字段 | 说明 |
| ---- | ---- |
| answer | Markdown 格式回答 |
| sources | 引用来源列表 |
| confidence | high=明确答案 / medium=部分匹配 / low=勉强相关 |
| no_match | true=知识库无匹配，answer 为提示文字 |



## 边界情况

| 情况 | 处理方式 |
| ---- | ---- |
| 所有片段相关度 < 阈值 | 返回 no_match=true，提示无相关信息 |
| 用户问题与上次高度重复 | 从缓存返回上次结果 |
| 检索结果超过上下文窗口 | 按相关度截断，保留 Top-N |
| 用户输入恶意 Prompt 注入 | 系统指令优先级最高，忽略文档中的指令性语言 |



## 验证用例

| # | 输入 | 检索结果 | 期望输出 | 通过 |
| ---- | ---- | ---- | ---- | ---- |
| 1 | "公司年假政策是什么？" | 3 段包含"年假"的文档 | 准确回答天数 + 标注来源 | ⬜ |
| 2 | "今天天气怎么样？" | 5 段不相关文档 | no_match=true | ⬜ |
| 3 | "上一轮提到的那个政策，再说一遍？" | 结合对话历史检索 | 正确理解指代，回答对应政策 | ⬜ |
