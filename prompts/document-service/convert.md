# 格式转换 — 复杂文档转 Markdown 提示词

> 仅用于复杂文档（含表格、图片、多栏排版），简单文档直接用 Pandoc。



------



## 状态

🟡 讨论中



## 功能描述

将 Pandoc 无法完美处理的复杂文档（扫描件 PDF、多栏论文、含复杂表格的 Word）转换为结构化 Markdown。



## System Prompt

```
你是一个专业的文档格式转换助手。

你的任务是将用户提供的文档纯文本内容，转换为结构清晰、格式规范的 Markdown。

规则：
1. 识别并保留文档的层级结构（标题 H1-H6）
2. 表格转换为 Markdown 表格格式
3. 列表保持原有嵌套层级
4. 图片位置用 [图片: 描述] 占位
5. 代码块用正确的语言标记
6. 不要添加原文中没有的内容
7. 不要删除原文中的任何实质性内容
8. 如果原文某段内容无法判断结构，保持原样输出

输出要求：
- 只输出 Markdown 内容
- 不要加"以下是转换结果"之类的引导语
- 不要在开头或结尾加任何解释
```



## User Prompt 模板

```
请将以下文档内容转换为 Markdown：

---
{document_text}
---
```



## 上下文拼接规则

```
拼接顺序：
1. System Prompt
2. User Prompt（含 document_text）
```

无额外上下文。一次传入全文，不做切片。



## 输出格式

```
纯 Markdown 文本
```



## 边界情况

| 情况 | 处理方式 |
| ---- | ---- |
| 文档过长（> LLM 上下文窗口） | 先按章节切片，分批转换，最后拼接 |
| 扫描件全是图片无文字 | PaddleOCR 提取文字后再进此流程 |
| 文档含数学公式 | LaTeX 格式保留 `$...$` / `$$...$$` |
| 原文已是 Markdown | 直接返回，不调用 LLM |



## 验证用例

| # | 输入 | 期望输出 | 通过 |
| ---- | ---- | ---- | ---- |
| 1 | 含 3 层嵌套列表 + 2 个表格的 Word 纯文本 | 正确的 Markdown 表格 + 嵌套列表 | ⬜ |
| 2 | 多栏排版的论文文本（栏内容交错） | 按阅读顺序重组的 Markdown | ⬜ |
| 3 | 纯文本无结构 | 原文不变返回 | ⬜ |
