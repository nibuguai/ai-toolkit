## 🎯 项目路线图（按难度重新编排）

### 第一阶梯：快速出成果，建立正反馈（1-2个月）
> 难度：⭐ - ⭐⭐  
> 目标：跑通 AI 模型调用流程，获得成就感

#### 项目1：TTS 文字配音服务（难度：⭐）
- **核心功能**：输入文字 → 选择音色 → 生成自然语音
- **进阶功能**：上传参考音频 → 克隆音色 → 用该音色读任意文字
- **技术方案**：
  - 基础 TTS：ChatTTS 开源模型
  - 音色克隆：GPT-SoVITS
  - 后端：Python FastAPI 封装为微服务
- **亮点**：效果惊艳，可集成到 QQ 机器人、LobeChat、象棋分析等所有项目
- **预计耗时**：1-2 天

#### 项目2：AI图像处理工具箱（难度：⭐⭐ - ⭐⭐⭐）

- **核心定位**：集成证件照制作、人像抠图、老照片修复、去水印、智能裁剪等高频图像处理需求，打造一个本地化、隐私安全的“图像处理百宝箱”。

- **功能模块规划**：

  **模块一：证件照专区**
  - 生活照 → 标准证件照：自动抠图，替换红/白/蓝背景
  - 证件照尺寸调整：生成1寸、2寸、签证照、护照照等标准尺寸
  - 证件照自动排版：单张或多张自动排版到5寸/6寸冲印版，导出可直接打印
  - 换正装（进阶）：用AI生成正装覆盖便装（可选，后期迭代）

  **模块二：人像抠图与背景处理**
  - 人像抠图：精准抠出人像，保留头发丝、半透明物体边缘（婚纱、纱裙等）
  - 照片换背景：将抠出的人像放到新背景（纯色、渐变色、自定义图片）
  - 透明背景生成：导出无背景PNG图片，用于海报、头像、证件照等二次创作

  **模块三：瑕疵去除与修复**
  - 去除路人/多余物体：旅游照、街拍中的多余人物、杂物一键消除
  - 去除水印/Logo/文字：图片上的水印、日期、文字一键清除
  - 老照片修复：模糊老照片清晰化 + 可选黑白上色

  **模块四：智能裁剪与拼接**
  - 智能构图裁剪：自动识别主体，按1:1、16:9、4:3、9:16等比例裁剪
  - 照片拼接：长截图拼接、九宫格、四宫格、自由拼图、横版拼接

  **模块五：图片批量处理**
  - 批量加水印、改尺寸、压缩、加白边、调色、格式转换

- **技术方案**：
  - 抠图：rembg / MODNet / SAM（Segment Anything Model）
  - 图像修复：Real-ESRGAN（清晰化）、CodeFormer（人脸修复）、LaMa（去除水印/路人）
  - 图像处理：Python Pillow / OpenCV
  - 后端：Python FastAPI（每个功能模块封装为独立接口）
  - 前端：Vue 3 可视化操作界面（拖拽上传、实时预览、参数调节、一键下载）

- **亮点**：
  - 所有数据处理在本地完成，主打隐私安全
  - 一个工具箱涵盖日常90%的图像处理需求
  - 展示多种AI模型的集成与工程化能力
  - 面试时可现场演示“生活照→证件照”全流程

- **预计耗时**：3-4 周（分模块迭代，先跑通证件照和抠图，再逐步加其他功能）

### 第二阶梯：能力整合，展示工程化思维（2-3个月）
> 难度：⭐⭐ - ⭐⭐⭐  
> 目标：把多个 AI 能力封装成独立微服务，用 Java 统一调度

#### 项目3：AI格式转换工厂（难度：⭐⭐⭐）
- **核心功能**：PDF ⇄ Word ⇄ Markdown 互相转换
- **技术方案**：
  - Word→PDF：LibreOffice 命令行
  - PDF→Word：pdf2docx
  - 任意格式→Markdown：Pandoc + 本地大模型（复杂文档用AI转换）
- **架构**：Java Spring Boot（业务层）+ Python FastAPI（AI层）
- **亮点**：完美结合Java和Python，展示微服务架构能力
- **预计耗时**：2-3 周

#### 项目4：本地私有知识库问答系统（RAG）（难度：⭐⭐⭐）
- **核心功能**：
  - 导入内部文档（PDF/Word/MD）
  - 文档切片 → 向量化 → 存入向量数据库
  - 用户提问 → 检索相关片段 → 大模型生成回答（严格基于私有资料）
- **技术方案**：
  - 向量化：bge-large-zh 等开源 Embedding 模型
  - 向量数据库：Chroma / Milvus
  - 大模型：本地 Qwen/Ollama 或 DeepSeek API
  - 框架：LangChain / LlamaIndex
- **亮点**：RAG 完整落地，回答可溯源，适合事业单位内部场景
- **预计耗时**：2-3 周

#### 项目5：视频字幕/语音转文字工具（难度：⭐⭐）
- **核心功能**：上传视频/音频 → 自动生成字幕文件（SRT/VTT）
- **技术方案**：OpenAI Whisper 开源模型
- **应用场景**：课程字幕提取、会议记录、语音备忘录
- **预计耗时**：3-5 天

#### 项目6：高铁 & 机票 MCP 查询助手（难度：⭐⭐）
- **核心功能**：
  - 高铁查询：自然语言问“明天北京到上海的高铁” → 返回车次、余票
  - 机票比价：查询多个平台 → 返回最便宜航班 + 平台比价
- **技术方案**：
  - 高铁：封装 12306 公开查询接口（参考开源 `mcp-12306`）
  - 机票：封装 Amadeus 免费 API 或 Skyscanner 非官方 API
  - 协议：MCP 协议包装，接入 LobeChat / QQ 机器人
- **亮点**：紧跟 MCP 前沿热点，展示 AI Agent 落地能力
- **预计耗时**：1 周

#### 项目7：网页签到小助手（难度：⭐⭐）
- **核心功能**：自动打开网页 → 登录 → 点击签到 → 截图保存
- **技术方案**：Python + Selenium（网页自动化框架）
- **学习点**：元素定位、等待机制、异常处理
- **亮点**：RPA 流程自动化入门，可扩展到数据抓取
- **预计耗时**：3-5 天

#### 项目8：中国象棋AI推理分析服务（难度：⭐⭐⭐⭐）
- **核心功能**：
  - 棋盘界面（Vue） → 用户摆棋 → AI分析最佳着法
  - 显示胜率曲线、后续变化推演、棋理解释
  - 复盘功能：上传棋谱 → 逐回合分析漏着/败着
  - 语音播报：集成 TTS 服务，用语音播报分析结果
- **技术方案**：
  - AI引擎：Pikafish（本地CPU推理）+ 可选LCZero（GPU神经网络评估）
  - 后端：Python FastAPI 调用引擎 + Java Spring Boot 业务层
  - 前端：Vue 象棋棋盘组件（参考 Chessground）
- **参考开源项目**：Pikafish + python-chinese-chess
- **亮点**：AI推理可视化，完美结合CPU+GPU，展示混合架构能力
- **预计耗时**：3-4 周

#### 项目9：自动化游戏助手（学术研究用途）（难度：⭐⭐⭐⭐）
- **核心功能**：模拟人类操作，自动完成单机种菜、签到等重复任务
- **技术方案**：
  - 感知层：OpenCV 模板匹配 + PaddleOCR 文字识别
  - 决策层：Python 状态机

- **学习目标**：深入理解 RPA 核心原理，掌握“感知-决策-执行”架构
- **注意**：仅限单机游戏学习，不涉及对抗性平台
- **预计耗时**：3-4 周
