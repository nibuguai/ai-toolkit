## 二、各层说明



### 2.1 用户端



| 端           | 方案                           | 说明                                           |
| ------------ | ------------------------------ | ---------------------------------------------- |
| 手机 APP     | uni-app 壳子（WebView）        | 仅加载远程 H5 地址，前端更新不需要重新打包 APP |
| H5 移动端    | 若依 Office 自带 UniApp 移动端 | 基于 UniApp + Vue3 + Wot Design Uni            |
| Web 管理后台 | 若依 Office 自带 Vue3 后台     | 基于 Vue3 + Vite + Element Plus                |



**APP 壳子的作用：** 一个 WebView 加载 `https://your-domain.com`，所有界面和逻辑由服务端 H5 提供。前端更新只需部署到 Nginx，用户下次打开自动生效，不需要重新安装 APP。



### 2.2 Nginx



唯一对外暴露的服务，职责：



- 托管前端 H5 静态文件
- 托管管理后台静态文件
- 将 `/api/*` 请求代理到 SpringBoot
- 不代理 FastAPI，FastAPI 对外不可访问



### 2.3 SpringBoot（若依 Office）



**选择若依 Office 作为后端底座，不从零开始。**



**复用若依 Office 已有能力：**



| 模块     | 说明                                         | 复用程度 |
| -------- | -------------------------------------------- | -------- |
| 系统管理 | 用户、角色、菜单、部门、字典、参数、通知     | 直接复用 |
| 系统监控 | 在线用户、定时任务、服务监控、缓存监控、日志 | 直接复用 |
| 系统工具 | 代码生成器、表单构建器、Swagger、文件管理    | 直接复用 |
| 移动端   | 登录注册、首页、消息通知、个人中心           | 直接复用 |
| AI 模块  | AI 对话界面、Prompt 管理、多模型切换         | 直接复用 |
| 知识库   | 文档上传、预览、分类、搜索                   | 直接复用 |



**你需要新增的模块：**



| 新增内容           | 说明                                                      |
| ------------------ | --------------------------------------------------------- |
| 文档处理接口       | 接收上传、创建任务、查询结果、下载文件                    |
| FastAPI 调用客户端 | 封装对 FastAPI 的 HTTP 调用，负责 Base64 编解码和结果存储 |
| 文件解析服务       | 从 PDF/Word 中提取纯文字（LLM 分析前需要）                |
| 额度管理           | 记录和扣减用户调用次数                                    |
| 前端页面           | OCR 页、转换页、分析页、结果页（在若依移动端基础上新增）  |



**可裁剪的模块：**



| 模块           | 建议     | 理由               |
| -------------- | -------- | ------------------ |
| HRM 人力资源   | 裁剪     | 与文档处理无关     |
| ERP 企业资源   | 裁剪     | 与文档处理无关     |
| 项目管理       | 裁剪     | 与文档处理无关     |
| 财务管理       | 裁剪     | 后期按需加回       |
| OA / BPM / CRM | 按需保留 | 面向企业客户时有用 |



### 2.4 FastAPI



**核心原则：** FastAPI 只做计算，不碰任何存储。



text

```
SpringBoot 做的事：
  上传文件 → 存到 MinIO → 读出来转 Base64 → 发给 FastAPI
  收到 FastAPI 结果 → 存到 MySQL / MinIO → 更新任务状态

FastAPI 做的事：
  接收 Base64 → 处理 → 返回结果
  不知道用户是谁，不访问任何数据库和存储
SpringBoot 做的事：
  上传文件 → 存到 MinIO → 读出来转 Base64 → 发给 FastAPI
  收到 FastAPI 结果 → 存到 MySQL / MinIO → 更新任务状态

FastAPI 做的事：
  接收 Base64 → 处理 → 返回结果
  不知道用户是谁，不访问任何数据库和存储
```



**SpringBoot 调用 FastAPI 的数据格式：**



| 场景     | SpringBoot 发送         | FastAPI 返回               |
| -------- | ----------------------- | -------------------------- |
| OCR      | 图片 Base64             | 识别文字                   |
| 文档转换 | 文件 Base64 + 文件名    | 转换后文件 Base64 + 文件名 |
| AI 分析  | 提取后的纯文字 + Prompt | 分析结果文字               |
| AI 对话  | 用户消息 + 对话历史     | AI 回复文字                |



**具体容器拆分与镜像制作详见《FastAPI 文档》。**

## 四、Docker 部署总览



### 4.1 容器清单



| 容器 | 镜像 | 对外端口 | 说明 |
| ---- | ---- | ---- | ---- |
| doc-nginx | nginx:alpine | 80, 443 | 唯一对外入口，托管前端 + 反向代理 |
| doc-springboot | 若依 Office 自建镜像 (JDK 21) | 无 | 业务中枢，仅内部访问 |
| doc-fastapi-vision | 自建 (python:3.11-slim) | 无 | 视觉服务（证件照、抠图、修复），需 GPU |
| doc-fastapi-language | 自建 (python:3.11-slim) | 无 | 语言服务（TTS、语音识别），需 GPU |
| doc-fastapi-document | 自建 (python:3.11-slim) | 无 | 文档服务（格式转换、RAG 知识库） |
| doc-fastapi-reasoning | 自建 (python:3.11-slim) | 无 | 推理服务（象棋 AI、RPA） |
| doc-mysql | mysql:8.0 | 无 | 用户数据、任务记录、对话历史 |
| doc-redis | redis:7-alpine | 无 | 会话缓存、任务队列 |
| doc-minio | minio/minio | 9001（可选） | 文件存储，管理端口供管理员使用 |

> **详细部署方案（含资源配置、健康检查、安全策略、备份方案等）请参阅《Docker 部署总览文档》。**



### 4.2 网络隔离



text

```
frontend 网络：Nginx ↔ SpringBoot

backend 网络：  SpringBoot ↔ FastAPI (全4个) ↔ MySQL ↔ Redis ↔ MinIO

SpringBoot 是 frontend 和 backend 网络之间的唯一桥梁

Nginx 和 FastAPI 不在同一网络，外部无法访问 FastAPI
```

> 详细网络配置（子网划分、防火墙规则、端口暴露策略等）请参阅《Docker 部署总览文档》。



### 4.3 数据卷



| 卷名 | 用途 |
| ---- | ---- |
| mysql-data | MySQL 数据持久化 |
| redis-data | Redis 数据持久化 |
| minio-data | 文件存储持久化 |
| model-cache | AI 模型文件持久化（OCR/Whisper 等，避免重建容器重新下载） |
| chroma-data | RAG 向量数据库持久化 |

> 详细卷配置（宿主机路径、备份策略、权限设置等）请参阅《Docker 部署总览文档》。
