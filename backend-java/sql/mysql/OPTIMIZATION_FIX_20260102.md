## 🔧 修复内容

### 1. 显示标题改为下拉选择 ✅

**问题描述**：
Switch 开关在某些情况下无法正常切换，用户体验不佳

**解决方案**：
- 将 Switch 开关改为 Select 下拉选择
- 选项明确：显示 / 不显示
- 更加直观和可靠

**代码修改**：
```vue
<!-- 之前：Switch 开关 -->
<Switch
  v-if="prop.type === 'boolean'"
  v-model:checked="formData[prop.key]"
  size="small"
  @change="handleUpdateConfig"
/>

<!-- 现在：Select 下拉选择 -->
<Select
  v-if="prop.type === 'boolean'"
  v-model:value="formData[prop.key]"
  :options="[
    { label: '显示', value: true },
    { label: '不显示', value: false }
  ]"
  class="w-full"
  @change="handleUpdateConfig"
/>
```

### 2. 彻底移除固定边距，真正实现0边距 ✅

**问题描述**：
即使将容器内边距设置为 0，画布周围仍有明显的空白区域，无法有效利用空间

**根本原因**：
1. `designer-canvas` 外层 div 有固定的 `p-4` 类（16px padding）
2. `GridLayout` 组件又额外设置了 padding
3. 双重边距导致即使设置为 0 也无效

**解决方案**：
```vue
<!-- ❌ 修复前：固定 padding -->
<div class="designer-canvas h-full w-full overflow-auto bg-gray-50 p-4">
  <GridLayout :style="{ padding: `${containerPadding}px` }">
  </GridLayout>
</div>

<!-- ✅ 修复后：只在外层容器使用动态 padding -->
<div 
  class="designer-canvas h-full w-full overflow-auto bg-gray-50"
  :style="{ padding: `${containerPadding}px` }"
>
  <GridLayout>
  </GridLayout>
</div>
```

**修改清单**：
1. 移除 `designer-canvas` div 的 `p-4` 固定类
2. 改为动态绑定 `padding: ${containerPadding}px`
3. 移除 GridLayout 上重复的 padding 设置
4. 确保 Page 组件无额外 margin
