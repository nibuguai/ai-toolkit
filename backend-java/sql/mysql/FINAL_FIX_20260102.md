### 问题2: 彻底解决边距问题，真正实现0边距 ✅

**问题描述**：
即使将容器内边距设置为 0，画布周围仍有大量空白，无法有效利用空间

**根本原因分析**：

发现了**三层**边距叠加：

1. **Page 组件默认边距**
2. **designer-canvas 外层 div 的固定 `p-4`** (16px padding)
3. **GridLayout 组件又设置了动态 padding**

总边距 = 默认边距 + 16px + 用户设置值 ❌

**解决方案**：

#### 第1步：强化 Page 组件零边距
```vue
<Page
  :content-class="{
    'p-0!': true,
    'm-0!': true,
  }"
  :content-style="{ 
    height: 'calc(100vh - 64px)', 
    padding: '0', 
    margin: '0' 
  }"
>
```

#### 第2步：移除 designer-canvas 固定边距
```vue
<!-- 之前 ❌ -->
<div class="designer-canvas h-full w-full overflow-auto bg-gray-50 p-4">

<!-- 现在 ✅ -->
<div 
  class="designer-canvas h-full w-full overflow-auto bg-gray-50"
  :style="{ padding: `${containerPadding}px` }"
>
```

#### 第3步：GridLayout 只设置间距，不设置边距
```vue
<!-- 之前 ❌ -->
<GridLayout
  :margin="[margin, margin]"
  :style="{ padding: `${containerPadding}px` }"
>

<!-- 现在 ✅ -->
<GridLayout
  :margin="[margin, margin]"
>
```

#### 第4步：同步修复渲染器
```vue
<!-- layout-renderer.vue -->
<div 
  class="layout-renderer h-full w-full"
  :style="{ 
    padding: layoutConfig.containerPadding 
      ? `${layoutConfig.containerPadding[0]}px` 
      : '10px' 
  }"
>
```

**现在的边距计算**：
```
总边距 = 用户设置值 ✅
```

## 📞 技术细节

### 边距层级（修复后）
```
Page (padding: 0, margin: 0)
  └─ designer-content (无额外边距)
      └─ designer-canvas (padding: ${containerPadding}px)
          └─ GridLayout (margin: [${margin}, ${margin}])
              └─ 组件
```

### 配置持久化
全局配置保存在第一个组件的 config 中：
```json
{
  "componentCode": "xxx",
  "config": {
    "showTitle": true,
    "title": "标题",
    "_globalMargin": 10,
    "_globalContainerPadding": 0
  }
}
```

前缀 `_global` 表示这是全局配置，不是组件配置。
