- **项目名称**: RuoYi Office 后端服务

- **构建工具**: Maven 3.8+

## 部署步骤

### 步骤1: 配置 SSH 免密登录

**在 Jenkins 服务器(140)上执行：**

```bash
# 进入 Jenkins 容器
docker exec -it jenkins bash

# 生成 SSH 密钥（如果已有密钥可跳过）
# 方式1: 使用默认名称 id_rsa
ssh-keygen -t rsa -b 4096 -C "jenkins@140" -f ~/.ssh/id_rsa -N ""

# 方式2: 使用自定义名称（如 jenkins_rsa）
# ssh-keygen -t rsa -b 4096 -C "jenkins@140" -f ~/.ssh/jenkins_rsa -N ""

# 将公钥复制到目标服务器
# 如果使用默认名称 id_rsa
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.30.130

# 如果使用自定义名称 jenkins_rsa
# ssh-copy-id -i ~/.ssh/jenkins_rsa.pub root@192.168.30.130

# 测试连接
# 如果使用默认私钥
ssh root@192.168.30.130 'echo "SSH连接成功"'

# 如果使用指定私钥
#ssh -i ~/.ssh/jenkins_rsa root@192.168.30.130 'echo "SSH连接成功"'

# 退出容器
exit
```

**说明**: `ssh-copy-id` 会将公钥添加到目标服务器的 `/root/.ssh/authorized_keys` 文件中。

**如果 `ssh-copy-id` 执行后仍然需要输入密码，请按以下步骤排查：**

#### 1.1 检查公钥是否已复制

```bash
# 在 Jenkins 容器中查看公钥内容
cat ~/.ssh/id_rsa.pub

# 在目标服务器(130)上检查 authorized_keys
ssh root@192.168.30.130 "cat ~/.ssh/authorized_keys"
```

**如果 authorized_keys 文件不存在或为空，手动复制公钥：**

```bash
# 方式1: 使用 ssh-copy-id（推荐，会自动处理权限）
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.30.130

# 方式2: 手动复制（如果方式1失败）
# 在 Jenkins 容器中执行
cat ~/.ssh/id_rsa.pub | ssh root@192.168.30.130 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

#### 1.2 检查目标服务器权限

**在目标服务器(130)上执行：**

```bash
# 检查 .ssh 目录权限（必须是 700）
ls -ld ~/.ssh
chmod 700 ~/.ssh

# 检查 authorized_keys 文件权限（必须是 600）
ls -l ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# 检查文件所有者（必须是 root）
ls -l ~/.ssh/authorized_keys
chown root:root ~/.ssh/authorized_keys
```

#### 1.3 检查 SSH 服务配置

**在目标服务器(130)上检查 SSH 配置：**

```bash
# 检查 SSH 配置
grep -E "PubkeyAuthentication|AuthorizedKeysFile|PasswordAuthentication" /etc/ssh/sshd_config

# 应该看到：
# PubkeyAuthentication yes
# AuthorizedKeysFile .ssh/authorized_keys
# PasswordAuthentication yes（可选，建议保留作为备用）

# 如果配置被禁用，修改后重启 SSH 服务
sed -i 's/#PubkeyAuthentication no/PubkeyAuthentication yes/' /etc/ssh/sshd_config
sed -i 's/PubkeyAuthentication no/PubkeyAuthentication yes/' /etc/ssh/sshd_config
systemctl restart sshd
```

#### 1.4 使用详细模式测试连接

**在 Jenkins 容器中执行：**

```bash
# 使用 -v 参数查看详细连接信息
ssh -v root@192.168.30.130 'echo "SSH连接成功"'

# 查看具体失败原因，常见问题：
# - Permission denied (publickey) - 公钥未正确配置
# - Host key verification failed - 需要添加 -o StrictHostKeyChecking=no
```

#### 1.5 验证免密登录

**成功配置后，应该不需要输入密码：**

```bash
# 测试连接（应该直接成功，不需要输入密码）
ssh -o StrictHostKeyChecking=no root@192.168.30.130 'echo "SSH免密登录成功"'
```

**从宿主机获取私钥**（用于 Jenkins 凭据配置）:

```bash
# Jenkins 容器挂载了 /opt/jenkins_home:/var/jenkins_home
# 在宿主机上查看私钥（根据实际密钥名称选择）

# 如果使用默认名称 id_rsa
cat /opt/jenkins_home/.ssh/id_rsa

# 如果使用自定义名称 jenkins_rsa
cat /opt/jenkins_home/.ssh/jenkins_rsa

# 或者从容器内查看
docker exec jenkins cat ~/.ssh/jenkins_rsa
```

### 步骤2: 在目标服务器部署 MySQL

**在目标服务器(192.168.30.130)上执行：**

**注意**: 如果目标服务器上还没有 MySQL Docker 镜像，需要先拉取镜像。`docker run` 命令会自动拉取镜像，但建议先手动拉取以验证网络连接。

#### 2.1 创建数据目录和配置

```bash
# 创建 MySQL 数据目录
mkdir -p /data/mysql/data
mkdir -p /data/mysql/conf
mkdir -p /data/mysql/init

# 创建 MySQL 配置文件
cat > /data/mysql/conf/my.cnf << 'EOF'
[mysqld]
# 基础配置
port=3306
datadir=/var/lib/mysql
socket=/var/run/mysqld/mysqld.sock
pid-file=/var/run/mysqld/mysqld.pid

# 字符集配置
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci

# 认证方式（兼容旧客户端）
default-authentication-plugin=mysql_native_password

# 大小写不敏感
lower_case_table_names=1

# 时间配置
explicit_defaults_for_timestamp=true

# 连接配置
max_connections=500
max_connect_errors=1000

# 缓存配置
innodb_buffer_pool_size=512M
innodb_log_file_size=256M

# 日志配置
slow_query_log=1
slow_query_log_file=/var/lib/mysql/slow.log
long_query_time=2

[client]
default-character-set=utf8mb4

[mysql]
default-character-set=utf8mb4
EOF
```

#### 2.2 配置 Docker 镜像加速器（推荐）

**如果无法从 Docker Hub 拉取镜像，建议先配置国内镜像加速器：**

**在目标服务器(192.168.30.130)上执行：**

```bash
# 创建或编辑 Docker daemon 配置文件
mkdir -p /etc/docker
cat > /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ]
}
EOF

# 重新加载 Docker 配置
systemctl daemon-reload
systemctl restart docker

# 验证配置是否生效
docker info | grep -A 10 "Registry Mirrors"
```

**说明**: 
- 配置镜像加速器后，`docker pull` 命令会自动使用加速器
- 如果某个镜像源不可用，Docker 会自动尝试下一个
- 配置后需要重启 Docker 服务

#### 2.4 启动 MySQL 容器

```bash
docker run -d \
  --name mysql \
  --restart always \
  -p 3306:3306 \
  -e TZ=Asia/Shanghai \
  -e MYSQL_ROOT_PASSWORD='Admin$123!' \
  -e MYSQL_DATABASE=ruoyi-office \
  -v /data/mysql/data:/var/lib/mysql \
  -v /data/mysql/conf/my.cnf:/etc/mysql/conf.d/my.cnf \
  -v /data/mysql/init:/docker-entrypoint-initdb.d \
  mysql:8.0.33
```

**说明**:

### 步骤3: 在目标服务器部署 Redis

**在目标服务器(192.168.30.130)上执行：**

**注意**: 如果目标服务器上还没有 Redis Docker 镜像，需要先拉取镜像。`docker run` 命令会自动拉取镜像，但建议先手动拉取以验证网络连接。

#### 3.1 创建 Redis 数据目录和配置

```bash
# 创建 Redis 数据目录
mkdir -p /data/redis/data
mkdir -p /data/redis/conf

# 创建 Redis 配置文件
cat > /data/redis/conf/redis.conf << 'EOF'
# 绑定地址（允许所有IP访问，生产环境建议限制）
bind 0.0.0.0

# 端口
port 6379

# 密码（生产环境强烈建议设置）
requirepass Admin$123!

# 持久化配置
appendonly yes
appendfsync everysec

# 内存限制（根据服务器内存调整）
maxmemory 512mb
maxmemory-policy allkeys-lru

# 日志级别
loglevel notice

# 数据目录
dir /data

# 后台运行（Docker 中不需要）
daemonize no

# 超时配置
timeout 300
tcp-keepalive 300
EOF
```

#### 3.2 拉取 Redis 镜像

## 故障排查

### SSH 免密登录失败

**问题**: 执行 `ssh root@192.168.30.130` 时仍然需要输入密码

**原因**: SSH 免密登录配置未成功，可能的原因：
1. 公钥未正确复制到目标服务器
2. 目标服务器文件权限不正确
3. SSH 服务配置禁用了公钥认证
4. 公钥格式错误

**排查步骤**:

#### 步骤1: 检查公钥是否已复制

```bash
# 在 Jenkins 容器中查看公钥
docker exec -it jenkins cat ~/.ssh/id_rsa.pub

# 在目标服务器上检查 authorized_keys
ssh root@192.168.30.130 "cat ~/.ssh/authorized_keys"
```

**如果 authorized_keys 为空或不存在，手动复制：**

```bash
# 在 Jenkins 容器中执行
docker exec -it jenkins bash
cat ~/.ssh/id_rsa.pub | ssh root@192.168.30.130 "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

#### 步骤2: 检查目标服务器权限

**在目标服务器(130)上执行：**

```bash
# 检查并修复权限
ls -ld ~/.ssh
chmod 700 ~/.ssh

ls -l ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
chown root:root ~/.ssh/authorized_keys
```

**正确的权限应该是：**
```
drwx------  root root  .ssh
-rw-------  root root  .ssh/authorized_keys
```

#### 步骤3: 检查 SSH 服务配置

**在目标服务器(130)上执行：**

```bash
# 检查 SSH 配置
grep -E "^PubkeyAuthentication|^AuthorizedKeysFile|^PasswordAuthentication" /etc/ssh/sshd_config

# 应该看到：
# PubkeyAuthentication yes
# AuthorizedKeysFile .ssh/authorized_keys

# 如果 PubkeyAuthentication 是 no，修改并重启
sed -i 's/^PubkeyAuthentication.*/PubkeyAuthentication yes/' /etc/ssh/sshd_config
sed -i 's/^#PubkeyAuthentication.*/PubkeyAuthentication yes/' /etc/ssh/sshd_config
systemctl restart sshd
```

#### 步骤4: 使用详细模式诊断

```bash
# 在 Jenkins 容器中执行，查看详细连接信息
docker exec -it jenkins ssh -v root@192.168.30.130 'echo "test"'

# 查看关键信息：
# - debug1: Offering public key - 说明正在尝试使用公钥
# - debug1: Server accepts key - 说明服务器接受了公钥
# - Permission denied (publickey) - 说明公钥认证失败
```

#### 步骤5: 验证免密登录

```bash
# 在 Jenkins 容器中测试（应该不需要输入密码）
docker exec -it jenkins ssh -o StrictHostKeyChecking=no root@192.168.30.130 'echo "SSH免密登录成功"'
```

**常见错误及解决方案**:

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `Permission denied (publickey)` | 公钥未配置或权限错误 | 检查 authorized_keys 和文件权限 |
| `Host key verification failed` | 首次连接需要确认主机密钥 | 使用 `-o StrictHostKeyChecking=no` |
| `Could not resolve hostname` | 网络不通或 DNS 问题 | 检查网络连接和 hosts 文件 |
| `Connection refused` | SSH 服务未运行 | 检查 `systemctl status sshd` |

### 密码包含特殊字符的处理

**问题**: 如果密码包含特殊字符（如 `!`、`$`、`&` 等），bash 可能会进行历史扩展或命令替换，导致命令执行失败。

**解决方案**:

1. **在命令行中使用单引号包裹密码**:
   ```bash
   # 错误示例（感叹号会被 bash 解释）
   docker exec -it mysql mysql -uroot -pYourStrongPassword123! -e "SELECT 1;"
   
   # 正确示例（使用单引号）
   docker exec -it mysql mysql -uroot -p'YourStrongPassword123!' -e "SELECT 1;"
   ```

2. **在 Docker 环境变量中使用单引号**:
   ```bash
   # 错误示例
   -e MYSQL_ROOT_PASSWORD=YourStrongPassword123!
   
   # 正确示例
   -e MYSQL_ROOT_PASSWORD='YourStrongPassword123!'
   ```

3. **在 SSH 命令中使用单引号包裹整个命令**:
   ```bash
   # 错误示例（感叹号会被 bash 解释）
   ssh root@192.168.30.130 "echo 'SSH连接成功!'"
   
   # 正确示例（使用单引号包裹整个命令）
   ssh root@192.168.30.130 'echo "SSH连接成功"'
   ```

4. **或者使用 -p 参数后不跟密码（会提示输入）**:
   ```bash
   # 交互式输入密码（最安全）
   docker exec -it mysql mysql -uroot -p -e "SELECT 1;"
   ```

## 性能优化

### 启动速度优化

#### 1. JVM 启动参数优化

已优化的 JVM 参数（加快启动速度）：
- `-XX:+UseG1GC` - 使用 G1 垃圾收集器（启动更快）
- `-XX:+UseStringDeduplication` - 字符串去重（减少内存占用）
- `-XX:MaxGCPauseMillis=200` - 最大 GC 暂停时间
- `-XX:+TieredCompilation` - 分层编译
- `-XX:TieredStopAtLevel=1` - 快速启动模式（减少编译时间）
- `-Dspring.jmx.enabled=false` - 禁用 JMX（减少启动开销）

#### 2. Maven 构建优化

Jenkinsfile 已优化：
- 使用并行构建 `-T 1C`（使用所有可用 CPU 核心）
- 增量构建（跳过不必要的 clean）
- 批处理模式减少输出

#### 3. Docker 构建优化

- 使用构建缓存 `--cache-from`
- 优化 Dockerfile 层结构

#### 4. 健康检查优化

- 减少等待时间：从 10 分钟减少到 3 分钟
- 检查间隔：从 10 秒减少到 5 秒
- 添加超时控制：curl 请求 3 秒超时

### JVM 参数优化（运行时性能）

根据服务器内存调整：

| 服务器内存 | 推荐 JVM 参数（运行时） |
|-----------|----------------------|
| 2GB | `-Xms256m -Xmx512m` |
| 4GB | `-Xms512m -Xmx1024m` |
| 8GB | `-Xms1g -Xmx2g` |
| 16GB | `-Xms2g -Xmx4g` |

**注意**：启动优化参数和运行时性能参数可能不同。启动时优先速度，运行时优先性能。

## 常见问题

### Q: 如何回滚到上一个版本？

```bash
# 1. 查看可用镜像
docker images | grep yudao-server

# 2. 停止当前容器
docker stop yudao-server
docker rm yudao-server

# 3. 使用上一个版本的镜像启动
docker run -d --name yudao-server ... yudao-server:previous_tag
```

### Q: 如何查看应用配置？

```bash
# 查看激活的配置文件
docker exec yudao-server env | grep SPRING_PROFILES_ACTIVE

# 查看 actuator 配置信息
curl http://192.168.30.130:48080/actuator/env
```

### Q: 如何修改配置不重新构建？

```bash
# 1. 挂载外部配置文件
docker run -d \
  -v /data/app/config/application-prod.yaml:/yudao-server/config/application-prod.yaml \
  ...

# 2. 使用环境变量覆盖
docker run -d \
  -e "SPRING_DATASOURCE_URL=jdbc:mysql://..." \
  ...
```
