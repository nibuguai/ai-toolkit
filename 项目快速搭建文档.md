## 三、后端搭建（基于 RuoYi-Vue-Plus）

### 1. 克隆并简化

bash

```
# 克隆最新 5.X 分支（支持 JDK 21）
git clone -b 5.X https://gitee.com/dromara/RuoYi-Vue-Plus.git backend-java
cd backend-java

# 删除不需要的模块（代码生成器）
rm -rf ruoyi-generator
```



编辑父 `pom.xml`，在 `<modules>` 中删除 `ruoyi-generator` 一行。

### 2. 修改基础配置

打开 `ruoyi-admin/src/main/resources/application-dev.yml`，修改数据库、Redis、MinIO 的连接信息（本地 Docker 会用默认端口）：

yaml

```
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ai_platform?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root123

  redis:
    host: localhost
    port: 6379

# 若依框架自带的文件存储，我们改用 MinIO
oss:
  type: minio
  endpoint: http://localhost:9000
  accessKey: minioadmin
  secretKey: minioadmin
  bucketName: ai-platform
```



### 3. 初始化数据库

创建数据库 `ai_platform`，字符集 `utf8mb4`，然后运行项目根目录下的 SQL 初始化脚本（通常在 `db/migration` 或 `sql` 目录，若依 Plus 可能通过 Flyway 自动建表，启动时自动运行）。

### 4. 验证

bash

```
# 在 backend-java 目录下
mvn clean install -DskipTests
cd ruoyi-admin
mvn spring-boot:run
```



访问 `http://localhost:8080`，若出现 JSON 响应（提示未登录）则成功。
管理后台默认账号：`admin`，密码：`admin123`。

## 四、管理端前端（若依自带）

若依自带的前端 `ruoyi-ui` 已在 `backend-java/ruoyi-ui` 目录中，可直接使用。为保持独立，我们把它拷贝到项目根目录。

bash

```
# 在 ai-platform 根目录执行
cp -r backend-java/ruoyi-ui frontend-admin
cd frontend-admin
npm install
npm run dev
```



打开 `http://localhost:80`（默认 Vite 端口可能为 5173），代理已配置到后端 `8080`，可直接登录管理后台。
如果你希望以后换成 Naive UI，可以后续操作，现在先用起来。

## 五、用户端前端（Vue3 + Naive UI）

使用精简模板 `vue-naive-admin` 裁剪出用户端。

### 1. 克隆并裁剪

bash

```
# 在 ai-platform 根目录执行
git clone https://github.com/jekip/vue-naive-admin.git frontend-user
cd frontend-user
rm -rf src/views/admin   # 删除管理端页面
```



### 2. 删掉路由中的管理端路径

编辑 `src/router/index.ts`，删除 `/admin` 开头的所有路由，只保留：

ts

```
routes: [
  { path: '/', name: 'Home', component: () => import('@/views/home/index.vue') },
  { path: '/login', name: 'Login', component: () => import('@/views/login/index.vue') },
  { path: '/register', name: 'Register', component: () => import('@/views/register/index.vue') },
]
```



### 3. 修改首页为工具卡片

编辑 `src/views/home/index.vue`，替换为：

vue

```
<template>
  <n-space vertical size="large">
    <n-h1 style="text-align: center">AI 工具箱</n-h1>
    <n-grid cols="3" x-gap="20" y-gap="20">
      <n-grid-item v-for="tool in tools" :key="tool.path">
        <n-card :title="tool.title" hoverable @click="$router.push(tool.path)">
          <p>{{ tool.desc }}</p>
        </n-card>
      </n-grid-item>
    </n-grid>
  </n-space>
</template>

<script setup lang="ts">
const tools = [
  { title: '证件照', desc: '智能抠图换背景', path: '/id-photo' },
  { title: '语音合成', desc: '文字转语音', path: '/tts' },
  { title: '知识库问答', desc: '文档检索与回答', path: '/rag' },
]
</script>
```



### 4. 配置代理到后端

编辑 `vite.config.ts`，添加：

ts

```
server: {
  proxy: {
    '/api': 'http://localhost:8080'
  }
}
```



### 5. 启动验证

bash

```
npm install
npm run dev
```



访问 `http://localhost:5173`，应看到工具卡片页面。点击任一卡片会跳转空白页（稍后填充功能）。

## 六、AI 视觉服务（FastAPI 空壳）

bash

```
mkdir -p services/vision-service/app/routers
cd services/vision-service
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install fastapi uvicorn
```



创建 `app/main.py`：

python

```
from fastapi import FastAPI
from app.routers import health

app = FastAPI()
app.include_router(health.router)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("app.main:app", host="0.0.0.0", port=8001, reload=True)
```



创建 `app/routers/health.py`：

python

```
from fastapi import APIRouter

router = APIRouter()

@router.get("/health")
def health_check():
    return {"status": "ok", "service": "vision-service"}
```



启动服务：

bash

```
uvicorn app.main:app --port 8001
```



访问 `http://localhost:8001/health` 验证。

### Dockerfile（可选）

dockerfile

```
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]
```



`requirements.txt` 内容：

text

```
fastapi==0.110.0
uvicorn[standard]
```

## 七、Docker Compose 一键编排

在 `ai-platform` 根目录创建 `docker-compose.yml`：

yaml

```
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: ai_platform
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    ports:
      - "9000:9000"
      - "9001:9001"
    volumes:
      - minio-data:/data

  spring-boot:
    build: ./backend-java/ruoyi-admin   # 需要准备 Dockerfile，或先用 maven 打包成 jar 直接用 image
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/ai_platform?useSSL=false&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root123
      SPRING_REDIS_HOST: redis
      MINIO_ENDPOINT: http://minio:9000
    depends_on:
      - mysql
      - redis
      - minio

  vision-service:
    build: ./services/vision-service
    ports:
      - "8001:8001"

volumes:
  mysql-data:
  minio-data:
```



> ⚠️ 为简化，Spring Boot 的 Docker 镜像需要你编写 Dockerfile 在 `backend-java/ruoyi-admin`，或先本地打成 jar 再用 `image: openjdk:21` + volumes 挂载。建议先用 `docker-compose up -d mysql redis minio`，然后本地 IDE 启动 Spring Boot，更灵活。

## 八、端到端验证

1. 启动所有基础设施：
   `docker-compose up -d mysql redis minio`
2. 启动后端：
   在 IDE 中运行 `ruoyi-admin` 里的 `Application`，或 `mvn spring-boot:run`
3. 启动管理端前端：
   `cd frontend-admin && npm run dev`
4. 启动用户端前端：
   `cd frontend-user && npm run dev`
5. 启动 vision 服务：
   `cd services/vision-service && uvicorn app.main:app --port 8001`

打开浏览器访问：

- 管理后台：`http://localhost:5173`（若依管理端）
- 用户工具站：`http://localhost:5174`（用户端，端口视具体 Vite 分配）
