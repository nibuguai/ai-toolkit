## Phase 1：视觉服务（预计 2-3 周）

> 归属：**vision-service**（doc-fastapi-vision:8181）
> 里程碑：生活照 → 标准证件照 全流程可演示
> 状态：⬜ 未开始



### Sprint 1.1：证件照核心链路（第一周）

- [ ] **智能抠图** `POST /api/v1/vision/id-photo/remove-bg`
  - 先跑通 rembg，后期可选换 SAM 提升精度
- [ ] **换纯色背景** `POST /api/v1/vision/id-photo/change-bg`
  - 红/白/蓝三色可选
- [ ] **标准尺寸裁剪** `POST /api/v1/vision/id-photo/resize`
  - 1寸/2寸/签证照/护照照
- [ ] **前端页面**：上传照片 → 选背景色 → 选尺寸 → 实时预览 → 下载
- [ ] SpringBoot 编排：证件照全流程（抠图 → 换背景 → 裁剪）

### Sprint 1.2：进阶功能（第二周）

- [ ] **冲印排版** `POST /api/v1/vision/id-photo/compose-sheet`
  - 多张证件照排版到 5寸/6寸版面
- [ ] **头发丝精抠优化**
  - 换用 MODNet 或 SAM 提升精度（对比 rembg 效果）

### Sprint 1.3：瑕疵修复（第三周）

- [ ] **去除水印** `POST /api/v1/vision/inpaint/remove-watermark`
  - LaMa 自动检测 + 去除
- [ ] **去除路人/物体** `POST /api/v1/vision/inpaint/remove-object`
  - 用户框选区域 → LaMa 填充
  - 前端 Canvas 选区工具
- [ ] **老照片修复** `POST /api/v1/vision/restore/old-photo`
  - Real-ESRGAN 清晰化 + DeOldify 上色（可选）

### Sprint 1.4：图像增强与批量处理（第三周末）

- [ ] **超分辨率放大** `POST /api/v1/vision/enhance/super-resolution`
  - Real-ESRGAN 2x/4x
- [ ] **人脸修复** `POST /api/v1/vision/enhance/face-restore`
  - CodeFormer 模糊人脸修复
- [ ] **批量处理** `POST /api/v1/vision/batch/process`
  - 批量压缩/加水印/改尺寸/格式转换
  - SpringBoot 异步任务队列

### Phase 1 完成标志

- [ ] 证件照全流程可演示（抠图 + 换背景 + 裁剪 + 排版）
- [ ] 至少 2 个修复功能可演示（去水印 / 老照片修复）
- [ ] SpringBoot 编排代码清晰，不是简单转发
- [ ] Docker Compose 一键启动

## Phase 2：语言服务（预计 2-3 周）

> 归属：**language-service**（doc-fastapi-language:8182）
> 里程碑：文字配音 + 音色克隆 + 语音转文字 + 实时变声 全流程可演示
> 状态：⬜ 未开始



### Sprint 2.1：TTS 文字配音

- [ ] **文字转语音** `POST /api/v1/language/tts/synthesize`
  - ChatTTS，多种预设音色可选
  - 前端提供"换一个音色"按钮
- [ ] **音色克隆** `POST /api/v1/language/tts/clone-voice`
  - GPT-SoVITS，上传 3-10 秒参考音频
  - 清晰无杂音的参考音频效果最佳

### Sprint 2.2：语音识别

- [ ] **音频转文字** `POST /api/v1/language/asr/transcribe`
  - Whisper（small/medium 模型），中英文自动检测
  - 长音频分片处理（>25分钟）
- [ ] **字幕生成** `POST /api/v1/language/asr/generate-subtitle`
  - 生成 SRT/VTT 字幕文件
  - 返回时间戳 + 文字段落

### 集成任务

- [ ] TTS 集成到 QQ 机器人（AstrBot）：文字回复 → 语音播报
- [ ] Whisper 集成到 QQ 机器人：语音消息 → 转文字 → 大模型回复

### Sprint 2.3：实时变声器（新增）

- [ ] **预设音色实现**（纯 DSP，无需 GPU）
  - 男声：音高 -5 半音 + 共振峰微调
  - 女声：音高 +7 半音 + 共振峰调整
  - 机器人：量化 + 环形调制
  - 花栗鼠：音高 +12 半音
  - 巨人：音高 -12 半音
  - 电话音：带通滤波 300-3400Hz
  - 空旷大厅：大混响
  - 耳语：降噪 + 气声增强
- [ ] **DSP 处理管线**
  - 预加重 → 基频提取 (pyworld) → 音高变换 (PSOLA) → 共振峰调整 → 效果器叠加
- [ ] **WebSocket 实时变声** `WS /api/v1/language/voice-changer/stream`
  - 双向音频流，每帧 20ms
  - 支持中途切换预设
  - 延迟 < 20ms
- [ ] **获取预设列表** `GET /api/v1/language/voice-changer/presets`
- [ ] **试听变声效果** `POST /api/v1/language/voice-changer/preview`
  - 上传音频片段 → 返回变声后音频
- [ ] **前端界面**：实时变声面板（选择预设 → 实时试听 → 音量/音调微调）

### Phase 2 完成标志

- [ ] 输入文字 → 生成语音，支持 3+ 种音色
- [ ] 上传参考音频 → 克隆音色 → 用该音色读新文本
- [ ] 上传音频/视频 → 生成 SRT 字幕文件
- [ ] 实时变声器可演示，8 种预设音色，延迟 < 20ms
- [ ] TTS 已集成到至少一个其他项目

## Phase 3：文档服务（预计 2-3 周）

> 归属：**document-service**（doc-fastapi-document:8183）
> 里程碑：上传企业文档 → 基于文档精准问答，回答可溯源
> 状态：⬜ 未开始



### Sprint 3.1：格式转换

- [ ] **Word / MD → PDF** `POST /api/v1/document/convert/to-pdf`
  - LibreOffice headless 模式转换
- [ ] **PDF → Word** `POST /api/v1/document/convert/to-word`
  - pdf2docx 库
- [ ] **任意格式 → Markdown** `POST /api/v1/document/convert/to-markdown`
  - Pandoc（简单文档）+ LLM（复杂文档/扫描件）
  - 扫描件 PDF → PaddleOCR → Markdown
- [ ] SpringBoot 层：文件类型校验、转换任务队列、结果缓存

### Sprint 3.2：RAG 知识库

- [ ] **文档上传与解析** `POST /api/v1/document/rag/upload`
  - LangChain DocumentLoaders（PDF/Word/Markdown）
  - 文本切片（512 token/块，重叠 50 token）
  - bge-large-zh 向量化（1024 维）
  - 存入 Chroma 向量数据库
- [ ] **检索问答** `POST /api/v1/document/rag/query`
  - 用户问题 → 向量化 → Chroma 相似度检索 Top-5
  - Prompt 拼接 → LLM 生成回答（标注引用来源）
  - 回答附带引用（第几份文档第几段）
- [ ] **知识库管理**
  - `GET /api/v1/document/rag/documents` — 文档列表
  - `DELETE /api/v1/document/rag/documents/{id}` — 删除文档
- [ ] SpringBoot 层：知识库增删改查管理、问答历史记录、并发控制

### Phase 3 完成标志

- [ ] PDF/Word/MD 互转，支持 5+ 种转换路径
- [ ] 上传 10+ 份文档到知识库 → 基于文档精准问答
- [ ] 回答附带引用来源
- [ ] RAG 检索 Top-5 命中率 > 80%

## Phase 4：推理服务（预计 3-4 周）

> 归属：**reasoning-service**（doc-fastapi-reasoning:8184）
> 里程碑：摆一个残局 → AI 分析最佳着法 + 胜率 + 语音播报
> 状态：⬜ 未开始



### Sprint 4.1：象棋 AI 分析

- [ ] **最佳着法分析** `POST /api/v1/reasoning/chess/analyze`
  - Pikafish 引擎（UCI 协议通信）
  - 返回：最佳着法 + 胜率 + 搜索深度
- [ ] **Board 界面**
  - Vue 棋盘组件（SVG/Canvas 绘制），支持手动摆棋
  - FEN 棋局字符串输入/输出
- [ ] **棋理解释**
  - 分析结果 → LLM 生成自然语言解释
  - 调用 Ollama/Qwen
- [ ] **语音播报**
  - 棋理文字 → language-service TTS 语音播报
- [ ] **棋谱复盘** `POST /api/v1/reasoning/chess/replay`
  - 上传 PGN/XQF 棋谱 → 逐回合分析漏着/败着
- [ ] **WebSocket 实时推送** `WS /api/v1/reasoning/chess/stream`
  - 搜索中实时推送胜率变化

### Sprint 4.2：RPA 自动化（可选）

- [ ] **网页签到** `POST /api/v1/reasoning/rpa/execute`
  - Selenium 自动化：打开网页 → 登录 → 签到 → 截图
- [ ] **定时触发**
  - SpringBoot `@Scheduled` 定时调用
- [ ] **任务管理**
  - `GET /api/v1/reasoning/rpa/tasks` — 任务列表
  - `GET /api/v1/reasoning/rpa/tasks/{id}/status` — 状态查询
- [ ] **游戏助手**（学术研究用途，可选）
  - 屏幕感知：OpenCV 模板匹配 + PaddleOCR
  - 决策引擎：Python 状态机
  - 执行层：PyAutoGUI（PC）/ ADB（模拟器）

### Phase 4 完成标志

- [ ] 棋盘界面可手动摆棋
- [ ] AI 分析返回最佳着法 + 胜率 + 文字棋理解释
- [ ] 棋理分析可语音播报（集成 TTS）
- [ ] 棋谱复盘功能可演示
